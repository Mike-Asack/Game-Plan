## 01. 게임 컨셉 및 개요

### 1.1 게임 컨셉

- **장르**: 카지노 / 클래식 카드 게임 / 1인칭 게임
- **플랫폼**: 웹 (반응형 - 데스크탑 / 모바일)
- **코어 컨셉**: 실제 카지노의 바카라 게임을 100% 표준 규칙으로 구현하여 사용자에게 리얼한 1인칭 카지노 경험 제공

### 1.2 게임 목표

- Player(플레이어) 또는 Banker(뱅커) 중 9에 가까운 쪽에 베팅하여 승리
- Tie(타이)에 베팅하여 높은 배당 획득 가능
- 점수판 분석을 통한 패턴 예측 및 전략적 베팅

### 1.3 게임 특징

- **8덱 고정 슈 시스템**: 416장 카드로 구성된 실제 카지노 환경
- **완전한 점수판 시스템** *(AI 구현에 어려움이 있습니다. 기존 구현 되어있는 시스템을 이용해야합니다)*
  - 빅로드 (대로)
  - 스몰로드 (소로)
  - 중국점 1군 (大眼仔)
  - 중국점 2군 (小路)
  - 중국점 3군 (甲由路)
  - 육매 (6점 표시)
- **프리게임 모드**: 점수판에 카드 기록을 찍고, 전략을 수립하기 위함.
- **실시간 카드 애니메이션**: 플립 효과 및 자연스러운 딜링 연출
- **반응형 디자인**: 모바일과 데스크탑 각각 최적화된 UI

### 1.4 기술 스택

- **프레임워크**: React + TypeScript
- **스타일링**: Tailwind CSS v4
- **애니메이션**: CSS Animation + React State
- **컴포넌트**: AllCasinoUI 라이브러리 활용

## 02. 플레이 루프

### 2.1 메인 게임 루프

```
[베팅 페이즈]
    ↓
사용자 베팅 (Player / Banker / Tie)
    ↓
베팅 확정 (BETTING 버튼 클릭)
    ↓
[딜링 페이즈]
    ↓
카드 배분 (Player 2장 → Banker 2장)
    ↓
3번째 카드 규칙 판정
    ↓
추가 카드 배분 (조건 충족 시)
    ↓
점수 계산 및 승자 결정
    ↓
[결과 페이즈]
    ↓
당첨 애니메이션 및 배당 지급
    ↓
10초 대기 (카드 표시 + 베팅 가능) < 이 부분 현재 AI가 받아드리기 힘든 부분이 있습니다.
해당 라운드의 결과를 10초간 지속하여 보여주되, 라운드는 종료된 상태이며,
유저는 새로운 라운드를 위하여 베팅이 가능합니다.
    ↓
카드 정리
    ↓
[베팅 페이즈로 복귀]
```

### 2.2 세부 타임라인 (1라운드)

| 시간 | 페이즈 | 액션 |
|------|--------|------|
| 0s | 베팅 | 사용자 베팅 입력 |
| ~Xs | 베팅 | 베팅 금액 조정 및 확정 |
| 0s | 딜링 시작 | Player Card 1 슬라이드 + 플립 (0.4s 완료) |
| 0.4s | 딜링 | Player Card 2 슬라이드 + 플립 (1.2s 완료) |
| 0.6s | 딜링 | Banker Card 1 슬라이드 + 플립 (0.8s 완료) |
| 1.0s | 딜링 | Banker Card 2 슬라이드 + 플립 (1.6s 완료) |
| 1.6s | 판정 | Natural (8/9) 체크 |
| 2.0s | 딜링 | 3번째 카드 규칙 적용 (필요 시) |
| ~3.5s | 결과 | 승자 결정 및 점수 표시 |
| 3.5s | 결과 | 당첨 팝업 표시 (3초) |
| 6.5s | 대기 | 카드 유지 + 베팅 가능 (10초) |
| 16.5s | 정리 | 카드 제거 및 베팅 페이즈 복귀 |

### 2.3 프리게임 모드

- **진입**: 중앙 FREE 버튼 클릭 (베팅 페이즈에만 활성화)
- **특징**:
  - 무료 1회 라운드 실행
  - 잔액 차감 없음
  - 실제 카드 결과 시뮬레이션
  - 점수판 기록됨
- **종료**: 자동으로 베팅 페이즈 복귀

| +/- 버튼 | 베팅 페이즈 | 딜링/결과 페이즈 |

## 04. 게임 특징 / 규칙 / 판정 로직

### 4.1 바카라 표준 규칙

> *(규칙 관련해서 에볼루션의 스피드바카라 및 일반 호텔 카지노의 바카라 룰을 수 차례 AI에 대입 및 체크 했으나 로직 상 더블체크가 필요한 부분입니다. 개발 적용하실때 필히 컨펌 요청 및 체크 부탁드리겠습니다.)*

#### 4.1.1 카드 값

| 카드 | 점수 |
|------|------|
| A | 1 |
| 2-9 | 카드 숫자 그대로 |
| 10, J, Q, K | 0 |

#### 4.1.2 점수 계산

- 카드 점수 합계의 **끝자리**만 사용 (모듈로 10)
- 예시: 7 + 8 = 15 → **5점**
- 최고 점수: **9점** (Natural 9)

#### 4.1.3 Natural 규칙

- Player 또는 Banker가 처음 2장으로 **8점 또는 9점**을 만들면 즉시 게임 종료
- Natural이 발생하면 3번째 카드 없음
- Natural 9 > Natural 8
- Natural 발생 시 특별 연출 (황금 배경 + NATURAL! 텍스트)

### 4.2 3번째 카드 드로우 규칙 (100% 표준 규칙)

#### 4.2.1 Player 3번째 카드 규칙

| Player 점수 | 액션 |
|-------------|------|
| 0-5 | 카드 1장 받음 |
| 6-7 | 스탠드 (받지 않음) |
| 8-9 | Natural (게임 종료) |

#### 4.2.2 Banker 3번째 카드 규칙

**Player가 3번째 카드를 받지 않은 경우:**

| Banker 점수 | 액션 |
|-------------|------|
| 0-5 | 카드 1장 받음 |
| 6-7 | 스탠드 |
| 8-9 | Natural |

**Player가 3번째 카드를 받은 경우:**

| Banker 점수 | Player 3번째 카드 값 | Banker 액션 |
|-------------|----------------------|-------------|
| 0-2 | 모든 카드 | 무조건 받음 |
| 3 | 0-7, 9 | 받음 |
| 3 | 8 | 스탠드 |
| 4 | 2-7 | 받음 |
| 4 | 0-1, 8-9 | 스탠드 |
| 5 | 4-7 | 받음 |
| 5 | 0-3, 8-9 | 스탠드 |
| 6 | 6-7 | 받음 |
| 6 | 0-5, 8-9 | 스탠드 |
| 7 | 모든 카드 | 스탠드 |
| 8-9 | - | Natural |

### 4.3 승패 판정

1. **점수 비교**: Player 점수 vs Banker 점수
2. **승자 결정**:
   - Player 점수 > Banker 점수 → **Player 승리**
   - Banker 점수 > Player 점수 → **Banker 승리**
   - 점수 동일 → **Tie (무승부)**

### 4.4 점수판 시스템

> *(개발시, 기존 점수판 시스템 활용 필수)*

#### 4.4.1 빅로드 (대로 / Big Road)

- **표시 방식**: 6행 × N열 그리드
- **기호**:
  - 🔵 파란 원 = Player 승리
  - 🔴 빨간 원 = Banker 승리
  - 🟢 녹색 원 = Tie (이전 결과에 겹쳐 표시)
- **진행 방향**:
  - 같은 결과 연속: 아래로 쌓임 (최대 6행)
  - 다른 결과: 오른쪽 새 열
  - 6행 초과: 오른쪽으로 이동 (용의 꼬리)

#### 4.4.2 스몰로드 (소로 / Small Road)

- **표시 방식**: 빅로드 하단에 배치
- **업데이트**: 빅로드와 동기화
- **용도**: 빅로드 요약 정보

#### 4.4.3 중국점 (大眼仔 / 小路 / 甲由路)

- **1군 (大眼仔)**: 빅로드 2열째부터 분석
- **2군 (小路)**: 빅로드 3열째부터 분석
- **3군 (甲由路)**: 빅로드 4열째부터 분석
- **판정 로직**:
  - 빅로드 패턴을 점수 기반으로 계산
  - 규칙성 있음 → 🔴 빨간 원
  - 규칙성 없음 → 🔵 파란 원
  - Tie는 별도 표기하지 않음

#### 4.4.4 육매 (6점 표시)

- **표시 방식**: 빅로드 원 내부에 작은 점 표시
- **조건**: Player 또는 Banker가 6점으로 승리한 경우
- **목적**: 6점 승리 빈도 추적 (패턴 분석)

### 4.5 슈 시스템

- **구성**: 8덱 × 52장 = **416장**
- **카드 분포**:
  - 각 숫자/모양: 32장씩 (8덱)
  - A: 32장
  - 2-9: 각 32장
  - 10, J, Q, K: 각 32장 (총 128장의 0점 카드)
- **슈 리셋**:
  - 카드 부족 시 자동 리셋
  - 수동 리셋 (슈 초기화 버튼)
  - 리셋 시 점수판 초기화

## 05. 보상 / 수치 테이블

> *(TORY 환수율 및 RTP 체크 필요 — 페이지 제목 주석)*

### 5.1 배당 테이블

| 베팅 타입 | 배당 | 설명 |
|-----------|------|------|
| **Player** | 1:1 | 베팅액의 100% 수익 |
| **Banker** | 1:0.95 | 베팅액의 95% 수익 (5% 커미션) |
| **Tie** | 1:8 | 베팅액의 800% 수익 |

### 5.2 배당 계산 예시

#### 예시 1: Player 베팅 $100 → Player 승리

- **총 베팅**: $100
- **배당**: $100 × 1 = $100
- **총 수령**: $100 (원금) + $100 (수익) = **$200**
- **순수익**: +$100

#### 예시 2: Banker 베팅 $100 → Banker 승리

- **총 베팅**: $100
- **배당**: $100 × 0.95 = $95
- **총 수령**: $100 (원금) + $95 (수익) = **$195**
- **순수익**: +$95

#### 예시 3: Tie 베팅 $10 → Tie 발생

- **총 베팅**: $10
- **배당**: $10 × 8 = $80
- **총 수령**: $10 (원금) + $80 (수익) = **$90**
- **순수익**: +$80

- ~~**Banker**: $50 손실~~

### 5.6 RTP (Return to Player)
